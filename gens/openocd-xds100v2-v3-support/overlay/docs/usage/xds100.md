# TI XDS100v2 and XDS100v3

The XDS100v2 and XDS100v3 are FTDI/MPSSE JTAG adapters. OpenOCD uses the
`ftdi` adapter driver and channel A for JTAG. The second FTDI channel is not
claimed by OpenOCD and can remain available for a board UART.

## USB identities

| Probe | VID:PID |
|---|---|
| XDS100v2 | `0403:a6d0` |
| XDS100v3 | `0403:a6d1` |
| Some embedded XDS100v2 designs | `0403:6010` |

Use the version-specific interface file whenever possible:

```console
openocd -f interface/ftdi/xds100v2.cfg -f target/<target>.cfg
openocd -f interface/ftdi/xds100v3.cfg -f target/<target>.cfg
```

`interface/ftdi/xds100.cfg` accepts both branded revisions and the generic
FT2232H identity. When several matching probes are connected, select one with:

```console
openocd -f interface/ftdi/xds100v2.cfg \
  -c "adapter serial <serial-number>" \
  -f target/<target>.cfg
```

## Programming

```console
openocd \
  -f interface/ftdi/xds100v2.cfg \
  -f target/<target>.cfg \
  -c "adapter speed 1000" \
  -c "program firmware.elf verify reset exit"
```

The target configuration and flash driver determine which image formats and
flash algorithms are available. The XDS100 layer supplies JTAG transport,
reset/EMU control, and the power-loss-latch recovery required before the first
JTAG chain examination.

## Target-only power cycle

If the target is power-cycled while OpenOCD and the probe remain connected,
run this in the OpenOCD telnet console before continuing:

```text
xds100_recover_after_target_power_cycle
```

## Linux permissions

Install `99-openocd-xds100.rules`, reload udev, reconnect the probe, and ensure
your account has access to the `plugdev` group where that group is used.

## Windows driver

The JTAG interface must be bound to a libusb-compatible driver used by the
OpenOCD/libftdi build. Preserve the second FTDI interface when it is used as a
UART. Record the original driver before changing it so TI tools can be restored.
