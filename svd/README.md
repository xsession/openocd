# CMSIS-SVD files

This directory contains generated CMSIS-SVD files for local debugger peripheral
views.

## Universal entry point

Use `tools/svd/openocd_svd.py` as the common front door for OpenOCD SVD work. It
validates committed SVDs, inventories `tcl/target` coverage, imports local
CMSIS-Pack archives, and delegates TI generation to the existing TI toolchain.

```powershell
python .\tools\svd\openocd_svd.py list
python .\tools\svd\openocd_svd.py validate svd
python .\tools\svd\openocd_svd.py inventory --format markdown --out svd\catalog.md
python .\tools\svd\openocd_svd.py import-pack --pack C:\packs\Vendor.Device_DFP.1.0.0.pack --vendor vendor
```

The tool is intentionally offline-friendly. It consumes local vendor packs and
installed vendor databases; it does not download or relicense vendor register
data.

## Microchip dsPIC

The top-level dsPIC SVD files were generated from the compressed Microchip EDC
descriptors in `open_microchip_tools`:

| SVD | Source pack |
|---|---|
| `dspic30f5011.svd` | `Microchip.dsPIC30F_DFP` 1.5.254 |
| `dspic33fj128mc802.svd` | `Microchip.dsPIC33F-GP-MC_DFP` 1.4.235 |
| `dspic33fj128mc804.svd` | `Microchip.dsPIC33F-GP-MC_DFP` 1.4.235 |
| `dspic33ep128gm604.svd` | `Microchip.dsPIC33E-GM-GP-MC-GU-MU_DFP` 1.6.297 |

The files were generated with `dspic-svd` 0.3.0. They pass its structural SVD
validator and its Cortex-Debug compatibility validator. The generator is
Apache-2.0 licensed; the register descriptions remain subject to Microchip's
pack license and are not relicensed as part of OpenOCD.

The local EDC revisions encode some reset values with unknown hexadecimal
nibbles. Those nibbles are represented as unknown bits in each SVD register's
reset mask rather than being treated as zero-valued known bits.

## Texas Instruments

TI SVD files live in `ti/` and are generated by
`tools/ti/c2000_toolchain` from the local CCS target database:

| SVD | Source |
|---|---|
| `ti/tms320f280049.svd` | `C:\ti\ccs2100\ccs\ccs_base\common\targetdb\devices\f280049.xml` |
| `ti/tms320f28069.svd` | `C:\ti\ccs2100\ccs\ccs_base\common\targetdb\devices\f28069.xml` |
| `ti/tms320f28m35x_c28x.svd` | `C:\ti\ccs2100\ccs\ccs_base\common\targetdb\devices\f28m35m52c1.xml` |
| `ti/tms320f28m35x_m3.svd` | `C:\ti\ccs2100\ccs\ccs_base\common\targetdb\devices\f28m35m52c1.xml` |

Regenerate the TI set from an installed CCS tree with:

```powershell
python .\tools\svd\openocd_svd.py generate-ti --ccs-root C:\ti\ccs2100 --skip-fetch
```

`--skip-fetch` keeps generation offline and skips CMSIS-pack based devices such
as MSPM0 unless an offline pack is provided.
