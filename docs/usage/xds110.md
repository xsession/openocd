# TI XDS110

The XDS110 is supported by OpenOCD's native `xds110` adapter driver. It can use
JTAG or SWD depending on the target configuration.

## Basic Usage

Use the TI wrapper:

```console
openocd -f interface/ti/xds110.cfg -f target/<target>.cfg
```

Or use the core interface file directly:

```console
openocd -f interface/xds110.cfg -f target/<target>.cfg
```

When more than one XDS110 is connected, select one by serial:

```console
openocd -f interface/ti/xds110.cfg \
  -c "adapter serial <serial-number>" \
  -f target/<target>.cfg
```

## Speed

Start conservatively on new hardware:

```console
openocd -f interface/ti/xds110.cfg \
  -c "adapter speed 1000" \
  -f target/<target>.cfg
```

The driver clamps unsupported speeds to the range supported by the probe and
firmware.

## Probe Commands

After `init`, the driver exposes:

```text
xds110 info
xds110 supply <millivolts>
xds110 supply 0
```

`xds110 supply` is only useful on probe variants that support target voltage
supply control. Use `0` to disable probe-supplied target voltage.

## Linux Permissions

Install the repository udev rules, reload them, and reconnect the probe:

```console
sudo cp udev/99-openocd-probes.rules /etc/udev/rules.d/
sudo udevadm control --reload-rules
sudo udevadm trigger
```

The XDS110 IDs covered by the rules are:

| VID:PID | Notes |
| --- | --- |
| `0451:bef3` | Common XDS110 debug probe identity. |
| `0451:bef4` | Alternate XDS110 identity. |
| `1cbe:02a5` | XDS110-compatible identity used by some configurations. |

Docker or WSL setups may still need explicit USB forwarding or
`--device /dev/bus/usb`.

## Windows Driver

The native XDS110 driver uses libusb. If OpenOCD reports `XDS110: failed to
connect`, check that the probe is visible to Windows and that the debug
interface is bound to a libusb-compatible driver such as WinUSB or libusbK.

Do not change unrelated serial or auxiliary interfaces unless you intend to
stop using them with TI tools.
